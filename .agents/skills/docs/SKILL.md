# Documentation Skill

This skill ensures that the project remains well-documented and accessible to all contributors.

## When to use this skill

- Use this when creating feature documentation in `docs/features/`.
- This is helpful for updating `README.md` or `INSTALLATION.md`.
- Use when documenting API endpoints (Swagger/OpenAPI).
- Use to maintain the changelog or technical references.

## How to use it

### Documentation Standards
- **Markdown**: Use GitHub-flavored markdown.
- **Structure**: Every feature needs a `DESIGN_PHILOSOPHY.md` and a `QA-REPORT.md`.
- **API**: Keep `backend/src/routes/v3_routes.js` documented with clear comments or JSDoc if possible.

### Artifacts
- Use the Brain artifacts for internal planning.
- Ensure all user-facing docs are concise and up-to-date.
- Link to specific file ranges when explaining complex logic.
