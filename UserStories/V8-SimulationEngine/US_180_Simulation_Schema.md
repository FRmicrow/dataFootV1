# 👤 US_180: Simulation Schema & Model Ledger
**Accountable Agent**: SQL Agent / Database Architect
**Feature Type**: Infrastructure / Database
**Mission**: Architect the storage layer for "The Forge," ensuring every model iteration, prediction snapshot, and telemetry event is perfectly traceable.

---

## 🎯 Strategic Objective
Enable definitive model comparison and institutional-grade data traceablity. This schema is the "Hard Drive" of the Quant Lab, storing every mathematical inference generated by the Forge.

## 📋 Instruction for SQL Agent
Please execute the following schema creation on `database.sqlite`. Ensure all foreign keys and constraints are strictly enforced to maintain referential integrity between models and their simulated results.

### �️ Technical Specification

```sql
-- 1. Create Model Registry (The Ledger of Intelligence)
CREATE TABLE IF NOT EXISTS V3_Model_Registry (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    model_name TEXT NOT NULL,
    version_tag TEXT NOT NULL UNIQUE,
    hyperparameters_json TEXT, -- JSON blob of XGBoost/RF config
    features_list_json TEXT,   -- Exact features included in this version
    trained_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    is_active INTEGER DEFAULT 0
);

-- 2. Create League Simulations (The Forge Session Tracker)
CREATE TABLE IF NOT EXISTS V3_Forge_Simulations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    league_id INTEGER NOT NULL,
    season_year INTEGER NOT NULL,
    model_id INTEGER,
    status TEXT CHECK(status IN ('PENDING', 'RUNNING', 'COMPLETED', 'FAILED')) DEFAULT 'PENDING',
    current_month TEXT,        -- Progress indicator: '2023-10'
    total_months INTEGER,
    completed_months INTEGER DEFAULT 0,
    summary_metrics_json TEXT, -- Brier, Log-Loss, ROI stored here upon completion
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (model_id) REFERENCES V3_Model_Registry(id)
);

-- 3. Create Simulation Results (The Atomic Inference Cache)
CREATE TABLE IF NOT EXISTS V3_Forge_Results (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    simulation_id INTEGER NOT NULL,
    fixture_id INTEGER NOT NULL,
    prob_home REAL,
    prob_draw REAL,
    prob_away REAL,
    predicted_score TEXT,      -- e.g., '2-1'
    actual_winner INTEGER,     -- 1, 0, 2
    is_correct INTEGER,
    edge_value REAL,           -- (Model % - Odds %)
    retrieved_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (simulation_id) REFERENCES V3_Forge_Simulations(id),
    UNIQUE(simulation_id, fixture_id)
);

-- 4. Create Performance Time-Series (For the Live Pulse Chart)
CREATE TABLE IF NOT EXISTS V3_Forge_Monthly_Metrics (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    simulation_id INTEGER NOT NULL,
    period_year_month TEXT NOT NULL, -- '2023-10'
    accuracy_rate REAL,
    brier_score REAL,
    roi_pct REAL,
    FOREIGN KEY (simulation_id) REFERENCES V3_Forge_Simulations(id)
);
```

## ✅ Acceptance Criteria
- Migration successfully creates the four (4) "Forge" tables.
- `V3_Model_Registry` prevents duplicate `version_tag` entries.
- `V3_Forge_Results` prevents duplicate `fixture_id` for the same simulation via its UNIQUE constraint.
- Foreign keys are active and functional.
