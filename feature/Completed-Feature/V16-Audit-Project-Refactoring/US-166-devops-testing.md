# US-166: Standardisation CI/CD et couverture de tests

**Rôle**: DevOps Engineer / QA Engineer

**Objectif**: En tant qu'Ingénieur DevOps, je veux automatiser la validation du code et garantir une livraison continue fiable.

**Tâches**:
- [ ] Auditer les pipelines CI/CD existants (GitHub Actions, Docker).
- [ ] Intégrer les tests unitaires et d'intégration dans le pipeline.
- [ ] Mettre en place des outils d'analyse de code statique (Lint, Sonar).
- [ ] Documenter les procédures de déploiement et de rollback.
- [ ] Augmenter la couverture de tests sur les parties critiques.

**Exigences**:
- Respecter les instructions de `devops-engineer.md` et `qa-engineer.md`.
- Utiliser des images Docker légères et sécurisées.

**Critères d'acceptation**:
- [ ] Le pipeline CI échoue en cas de tests KO ou de régression de style.
- [ ] Les déploiements sont reproductibles en un clic.
- [ ] La couverture de tests est jugée satisfaisante selon la `review-checklist.md`.
