# US-270 : Refonte de la Navbar avec le Design System

**1. Contexte :**
- Feature parente : V27 - FixFrontIssue
- Couche technique ciblée : Frontend

**2. Intention (Qui & Quoi) :**
En tant qu'utilisateur, je veux que la barre de navigation (Navbar) utilise les composants standards du Design System (notamment les boutons) afin d'offrir une expérience cohérente avec le reste de l'application V3.

**3. Raison (Pourquoi) :**
Afin d'améliorer la qualité du frontend et d'harmoniser l'interface globale.

**4. Détails / Scénarios (Critères d'Acceptation) :**
- [ ] Les boutons de la Navbar utilisent les composants du design system (`src/design-system/Button.jsx` ou équivalent).
- [ ] L'affichage et le responsive de la Navbar restent harmonieux et fonctionnels après intégration.
- [ ] Aucun composant de base UI ne doit être recréé si une alternative existe dans le Design System.
